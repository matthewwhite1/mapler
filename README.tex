\section{mapler}\label{mapler}

\href{https://github.com/matthewwhite1/mapler/actions/workflows/R-CMD-check.yaml}{\pandocbounded{\includesvg[keepaspectratio]{https://github.com/matthewwhite1/mapler/actions/workflows/R-CMD-check.yaml/badge.svg}}}

The goal of mapler is to provide functions to investigate how maple
syrup production could be impacted by climate change.

\subsection{Downloading Climate Data}\label{downloading-climate-data}

This package provides three functions for downloading climate data:
\texttt{download\_ghcnd()}, \texttt{download\_loca2()}, and
\texttt{download\_prism()}. These functions are useful for downloading
both historic climate data and future climate projections.

Once data is downloaded, it can be loaded into R using
\texttt{loca\_t\_rast()} or \texttt{prism\_t\_rast()}, ``t'' meaning
temperature. These functions take filepaths of where the data is
downloaded and return the tmin and tmax raster stacks.

\subsection{Calculating Sap Days}\label{calculating-sap-days}

Maple sap tapping is highly dependent on the freeze/thaw cycle. This
package calls a ``sap day'' a day where the temperature switched between
freezing and thawing during the day.

The function \texttt{sap\_day()} takes in tmin and tmax raster stacks
and aggregates them into a yearly sap day stack. This combination of
tmin and tmax is used to calculate the proportion of sap days in a year
at every pixel. This function can be used for future climate projection
with LOCA2 data.

The \texttt{sap\_day()} function can be used with LOCA2 data like so:

\subsection{Package Data}\label{package-data}

This package contains a few datasets from various sources. The
\texttt{farms} datasets (\texttt{us\_farms}, \texttt{canada\_farms},
\texttt{quebec\_farms}) contain maple farm names and addresses scraped
from the internet. The \texttt{farms\_coords} dataset contains geocoded
latitude and longitude coordinates of these farms.

This package also contains two datasets to assist in downloading climate
data: \texttt{loca2\_model\_names} and \texttt{ghcnd\_stations}.

For an example of what \texttt{sap\_day()} can do, this package contains
an example raster called \texttt{test\_loca\_sap\_day}. This was created
by downloading historical and scenario ssp585 tmin and tmax data from
the LOCA2 model ACCESS-CM2, loading the rasters into R using
\texttt{loca\_t\_rast()}, calculating yearly sap day proportions using
\texttt{sap\_day()}, shifting the longitude by -360 to match WGS84, and
aggregating by a factor of 20 to decrease the file size. It can be
accessed like so:

\subsection{Future Climate Projection}\label{future-climate-projection}

The \texttt{sens} family of functions can be used for future climate
projection. Output from \texttt{sap\_day()} can be passed into
\texttt{sens\_farms()}, along with farms coordinates. This function
combines the \texttt{get\_sens\_farms()} and
\texttt{get\_sens\_joined()} functions. \texttt{sens\_farms()} starts by
calculating the Sen's slope for each farm and creating a new variable
that is 1 if the slope is significant positive, -1 if the slope is
significant negative, and 0 otherwise. Then, the farms are joined to a
shapefile, and a proportion of the significant variable is calculated
for each group in the shapefile. This resulting dataframe can be plotted
with something like ggplot.
